## 5. Mission Control Enrichment Workflow

### The Enrichment Loop

Mission Control (the Modex RevOps app) has the data infrastructure to enrich any account from "persona list" to "demo-grade showcase microsite." Here is the exact workflow:

### Step 1: Research (Source: external)
**Input:** Account name + vertical
**Action:** Build a dossier using LinkedIn, SEC filings, earnings calls, press releases, MODEX exhibitor list, industry news
**Output fields needed:**
- `facility_count`, `facility_types`, `daily_trailer_moves`, `fleet_profile`
- `specific_pain_points[]` (3 minimum, each with headline + description + kpiImpact)
- `custom_hero` (subheadline - prospect-facing, no internal language)
- `annual_revenue`
- MODEX/event signals: `modex_speaker_name`, `modex_speaker_title`, or `new_exec_name`, `new_exec_title`

**Where it goes:** `src/lib/data/accounts.json` - direct field update

### Step 2: Person Research (Source: external + all-contacts.csv)
**Input:** Account name
**Action:** Cross-reference all-contacts.csv for existing contacts at account domain. Research LinkedIn for career history, recent promotions, mandate signals.
**Output fields needed per person:**
- `person_hero_overrides` - Record<person name, custom headline> in accounts.json
- Optionally: persona career intel for the dossier

**Where it goes:** `src/lib/data/accounts.json` `person_hero_overrides` field + optionally `src/lib/data/personas.json` role_in_deal/seniority updates

### Step 3: Proof + Module Customization (Source: internal knowledge)
**Input:** Account vertical + specific pain points
**Action:** Write custom proof quote attribution and module relevance descriptions
**Output fields needed:**
- `custom_proof_quote` + `custom_proof_attribution` (or reuse closest vertical match)
- `custom_module_relevance` - Record<"flowDRIVER"|"flowBOL"|"flowSPOTTER"|"flowTWIN", description>

**Where it goes:** `src/lib/data/accounts.json`

### Step 4: Regenerate + Verify
**Action:** Run `npx tsx scripts/generate-microsite-data.ts`
**Verify:** Open `/for/[account-slug]` and `/for/[account-slug]/[person-slug]` - check:
- [ ] Hero headline is prospect-facing (no internal language)
- [ ] Subheadline is prospect-facing (no "YardFlow story should translate well")
- [ ] Module relevance descriptions are specific (not "Standardizes X across...")
- [ ] Person variant has custom hero (not "the yard is where [Company]'s execution breaks down")
- [ ] Proof quote is present and attributed
- [ ] Signal bar text is prospect-facing
- [ ] Accent color is correct and unique

### Step 5: Screenshot + Promote
**Action:** Capture screenshot, set `showcase: true` and `showcase_order` in accounts.json
**Validate:** Run `npm run build` to verify no Railway/build errors

### Enrichment Time Estimate Per Account
- **From scratch (no data):** ~2 hours (research + writing + generation + QA)
- **From dossier (has research):** ~45 minutes (writing + generation + QA)
- **From enriched non-showcase (has data, needs overrides):** ~30 minutes (overrides + generation + QA)

### Where Mission Control Adds Leverage
The app already has:
- `all-contacts.csv` with 102,542 contacts searchable by domain
- Persona records with emails, titles, roles for all 15 accounts
- Research dossiers for 8 companies (Toyota, Mondelez, Campbell's, AB InBev, Ford, Constellation, KDP, General Mills)
- `docs/research-dossiers-top10.md` with consolidated research on top targets
- Generator that automatically builds microsites from enriched account data
- Activity logging that tracks every page view with account + persona + timestamp

**Mission Control's highest leverage is NOT building new features. It's feeding the generator better data.** Every hour spent on account research produces a measurably better microsite. Every hour spent on building features produces marginal UI improvement.
